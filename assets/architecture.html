CSRS — Cybersecurity Standards RAG System

Fully offline retrieval-augmented generation over cybersecurity standards. 4 documents, 2506 chunks, 532 pages.

Ingest — Pipeline.index()

  • • docs/ scanned recursively for .pdf and .txt
  • • SHA-256 of the file bytes gates the work, so an unchanged file is skipped before the parser opens it
  • • Docling runs a layout model that classifies page headers and footers structurally
  • • 400-token chunks split at headings first, carrying page, section breadcrumb and control ID
  • • Cold build 316 s; unchanged reload 0.057 s

Query — Pipeline.ask()

  • • nomic-embed-text with the search_query: task prefix
  • • Chroma cosine search, top 5 of 2506 chunks
  • • The prompt contains the retrieved chunks and nothing else about the corpus
  • • A grounding instruction closes the prompt; refusal is detected by exact match
  • • Answers stream token by token over NDJSON

Offline guarantee

  • • Ollama on http://127.0.0.1:11434 is the only model runtime
  • • Six models, about 14 GB, plus 1.3 GB of Docling weights
  • • 133 offline tests pass with Ollama pointed at a dead port
  • • Web fonts vendored as local .woff2; no CDN at runtime
  • • No cloud LLM client exists anywhere in src/
